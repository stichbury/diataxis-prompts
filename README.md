Start by clearly stating what problem this guide solves ("This guide shows you how to..."). Focus entirely on the practical steps needed to achieve the goal - remove any teaching or background explanation. Structure the content around the user's workflow, not the tool's features. Address real-world complexity by showing how to adapt the guidance to different situations. Cut any unnecessary completeness - start and end at reasonable points, assuming the user can connect this to their broader work. Eliminate any reference to learning or skill-building - this is about getting work done efficiently.

## Reference Prompt

Rewrite this text as **reference material** that provides neutral, factual information. Remove all personal pronouns and instructional language. Present information in a structured, systematic way that mirrors how the software itself is organized. State facts about what the software does, not how to use it ("X function accepts Y parameters and returns Z"). List all options, parameters, commands, and features comprehensively. Include warnings where appropriate ("You must use A. Never do B."). Remove all opinions, explanations of why, and subjective judgments. Make the content austere and authoritative - something a user would quickly consult to check a specific fact, not read through. Organize information so users can find exactly what they need without reading everything else.
